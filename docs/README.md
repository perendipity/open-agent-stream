## Structure

- `/vision`: why OAS exists, what portability means here, and what the project
  will not become
- `/architecture`: how session data is gathered, retained, replayed, and
  delivered
- `/governance`: how the standard evolves
- `/integrations`: third-party adapter/sink authoring guidance
- `/adoption`: install guidance and operating notes for one-machine and
  multi-machine use
- `/packaging`: service templates for released binaries
- `/roadmap`: release framing and adoption criteria
